Pentru prezentare Rails trebuie scoase in evidenta urmatoarele aspecte:
  - faptul ca Rails a popularizat limbajul de programare ruby
  - arhitectura MVC
  - principiile de inginerie software folosite:
    - convention over configuration (CoC http://en.wikipedia.org/wiki/Convention_over_configuration)
    - don't repeat yourself (DRY http://en.wikipedia.org/wiki/Don%27t_repeat_yourself)
    - active record pattern (http://en.wikipedia.org/wiki/Active_record_pattern)
      - aici poti include si ORM-ul (!!!Object-relational mapping http://en.wikipedia.org/wiki/Object-relational_mapping !!! numai pentru documentatia ta)
        din Rails ActiveRecord
    - "Fat models, skinny controllers" means that most of the application logic should be placed within the model while leaving the controller as light as possible.
  - prototipare rapida cu tehnica de generare ( scaffolding )
